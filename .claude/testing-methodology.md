# Testing Methodology

## 테스트 실행 방식

[TM-01] 공통 인터페이스를 통해 각 데이터베이스를 순차적으로 실행
[TM-02] 실제 데이터베이스 입출력을 테스트 (파일 시스템 I/O 포함)
[TM-03] 각 데이터베이스 테스트 후 메모리 정리 수행
[TM-04] 각 데이터베이스 테스트 후 DB 파일 및 데이터 완전 삭제

## 데이터 양 증가 패턴

[TM-05] 데이터 양을 1K->10K->100K 단계로 증가시키며 테스트
[TM-05-1] 각 단계는 이전 단계의 1/10 단위로 추가 (1K 생성 -> 9K 추가 -> 90K 추가)
[TM-06] 각 단계마다 TM-09~TM-18 테스트 시나리오 수행 (Delete 제외)
[TM-06-1] 모든 데이터 단계 완료 후 마지막에 TM-19 Delete 테스트 1회만 수행
[TM-07] 이전 테스트 데이터를 유지하여 데이터 의존성 있는 테스트 가능
[TM-08] DB 전환 시에만 데이터 제거, 동일 DB 내에서는 데이터 누적

## 테스트 시나리오

[TM-09] 초기 로딩: DB 초기화 및 첫 접근 시간 측정
[TM-10] Create: 개별 데이터를 하나씩 순차적으로 생성, 각 데이터는 고유한 값 보유
[TM-11] 배치 작업: 여러 데이터를 한 번에 일괄 생성 (트랜잭션 또는 배치 API 활용)
[TM-12] Read: Primary Key 또는 ID 기준 단순 조회 수행
[TM-13] 색인 검색: 인덱스가 적용된 필드 기준으로 조건 검색 수행
[TM-14] 비색인 검색: 인덱스가 없는 필드 기준으로 조건 검색 수행 (Full Scan)
[TM-15] 쿼리 복잡도별 성능: 단순 조회, 다중 조건, 집계 연산 등 복잡한 쿼리 수행
[TM-16] Update: 기존 데이터의 특정 필드 값 수정
[TM-17] 트랜잭션 처리: 트랜잭션 내에서 CRUD 복수 작업 수행 후 커밋
[TM-18] 동시성 처리: 여러 스레드/태스크에서 동시에 읽기/쓰기 작업 수행
[TM-19] Delete: 기존 데이터 삭제 (모든 데이터 단계 완료 후 마지막에만 1회 수행)

데이터 단계별 테스트 실행 순서:
1. 1K 데이터: TM-09 -> TM-10 -> TM-11 -> TM-12 -> TM-13 -> TM-14 -> TM-15 -> TM-16 -> TM-17 -> TM-18
2. 10K 데이터 (9K 추가): TM-10 -> TM-11 -> TM-12 -> TM-13 -> TM-14 -> TM-15 -> TM-16 -> TM-17 -> TM-18
3. 100K 데이터 (90K 추가): TM-10 -> TM-11 -> TM-12 -> TM-13 -> TM-14 -> TM-15 -> TM-16 -> TM-17 -> TM-18
4. 최종 Delete 테스트: TM-19 (1회만 수행)

## 측정 방법 및 정확도

[TM-20] 각 데이터베이스당 1회씩만 측정
[TM-21] 시간 측정 단위는 밀리초(ms)
[TM-22] 데이터 준비 시간 제외, 순수 DB 작업 시간만 측정
[TM-23] 단일 값만 기록 (평균/중앙값 없음)
[TM-24] ContinuousClock 사용하여 시간 측정 (Swift 6.0 표준, 시스템 변경 영향 없음, 타입 안전)
[TM-25] Instruments를 보조 도구로 사용하여 메모리, CPU 사용률 측정 및 결과 검증

## 결과 파일 형식 및 저장

[TM-26] JSON 포맷으로 결과 저장
[TM-27] 저장 위치는 프로젝트 내 특정 폴더 (Results/)
[TM-28] 파일 네이밍 규칙: {timestamp}-{database_name}.json
[TM-29] 포함할 메타데이터:
- 테스트 일시 (ISO 8601 형식)
- 데이터베이스 이름 및 버전
- 환경 정보 (CPU 사용률, 메모리 상태, 디스크 상태, 백그라운드 프로세스)
- 데이터 양 정보 (1K, 10K, 100K)
- 각 테스트 시나리오별 측정 결과 (TM-09 ~ TM-19)
